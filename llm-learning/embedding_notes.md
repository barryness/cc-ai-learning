## 1. 起点问题：为什么 AI 知道同义词

### 核心问题

当你在搜索引擎输入"汽车"，它也会返回"轿车""SUV""跑车"的内容。

它怎么知道这些词**意思接近**？

### 直觉类比

想象你给世界上所有人发了一张问卷：

> 在"汽车"出现的位置，还能填哪些词？

大家填写："轿车""跑车""SUV""巴士""卡车"...

> 在"香蕉"出现的位置，还能填哪些词？

大家填写："苹果""橘子""水果""菠萝"...

**规律**：意思接近的词，出现在**相似的上下文中**。

### 这就是 Embedding 的核心思想

```
"汽车"和"轿车" → 经常出现在相同的上下文中 → 意义接近
"汽车"和"香蕉" → 几乎不出现在相同的上下文中 → 意义不接近
```

Embedding 把这种"意义"转换成了**数字**，让计算机能处理。

## 2. Embedding 是什么

### 通俗解释

**Embedding = 把词语/句子/文档，转换成一组数字的列表（向量）。**

```
文字                                 数字向量
"汽车"    ──→  [0.23, -0.45, 0.78, 0.12, -0.33, ...]
"轿车"    ──→  [0.21, -0.43, 0.75, 0.14, -0.31, ...]
"香蕉"    ──→  [-0.67, 0.89, -0.12, -0.55, 0.41, ...]
```

关键：**意思接近的词 → 向量也接近。意思不接近的词 → 向量也不接近。**

就像把文字"翻译"成一种**机器能计算的数字语言**。

### 技术解释

Embedding 模型是一个神经网络，训练目标很简单：

> 给定一个词，预测它的上下文词（周围的词）。或者反过来：给定上下文，预测中间的词。

训练方式（以 Word2Vec 为例）：

```
1. 喂入海量文本（维基百科、新闻、书籍）
2. 对每个词，创建一个随机向量作为"初始值"
3. 让模型做预测：看到"汽车"，猜周围有哪些词
4. 预测"轿车"的概率应该很高，预测"香蕉"的概率应该很低
5. 根据预测误差，调整向量值
6. 重复几亿次
```

最终，每个词被嵌入到一个**高维空间**中（通常 384 维或 768 维）。语义相近的词在这个空间中**距离很近**。

### 实际代码

```python
from sentence_transformers import SentenceTransformer

# 加载模型（只下载一次，约 80MB）
model = SentenceTransformer('all-MiniLM-L6-v2')

# 把文字变成向量
embeddings = model.encode(['汽车', '轿车', '香蕉'])
# 输出: (3, 384) 的矩阵——3个词，每个词是384个数字的向量

print(embeddings.shape)  # (3, 384)
print(embeddings[0][:5]) # 汽车的前5个维度：[0.045, -0.032, ...]
```

## 3. 向量是什么

### 通俗解释

**向量 = 一组有序的数字。** 每个数字代表一个"维度"上的位置。

```
一维向量：[3]         → 数轴上的一个点
二维向量：[3, 4]      → 平面上的一个点
三维向量：[3, 4, 5]   → 空间中的一个点
384维向量：[...384个数字...] → 高维空间中的一个点
```

### 核心直觉

把向量想象成**坐标**：

```
[3, 4]  →  从原点出发，向右走3步，向上走4步
[1, 2]  →  从原点出发，向右走1步，向上走2步
```

两个向量位置越近 → 它们代表的东西越相似。

### 在 Embedding 中

每个维度没有明确的名字（不是"颜色""大小""价格"这种），它是在训练过程中自然学到的。

但我们可以这样**类比理解**：

```
假设（简化）每个 Embedding 只有 3 个维度：

维度1：约等于"是否跟交通有关"
维度2：约等于"是否是生物"
维度3：约等于"是否是人造物"

"汽车" → [0.9, 0.1, 0.8]    # 交通相关、非生物、人造物
"轿车" → [0.8, 0.1, 0.7]    # 和"汽车"很接近
"猫"   → [0.1, 0.9, 0.1]    # 非交通、生物、非人造物
```

真实的 384 个维度学到的特征更复杂、更抽象，但原理一样。

## 4. 相似度是什么

### 通俗解释

**相似度 = 两个向量"有多接近"的量化指标。**

在 Embedding 空间中：

```
  "汽车" 的点 ← 近 → "轿车" 的点      → 相似度高
  "汽车" 的点 ← 远 → "香蕉" 的点      → 相似度低
```

### 怎么量化"近"和"远"

有几种计算方式：

**欧几里得距离**（直线距离）：
```python
distance = np.sqrt(np.sum((vec1 - vec2) ** 2))
# 距离越小 → 越相似
```

**余弦相似度**（夹角的余弦值）：
```python
cosine = np.dot(vec1, vec2) / (np.linalg.norm(vec1) * np.linalg.norm(vec2))
# 值越接近 1.0 → 越相似
# 值越接近 0   → 不相关
# 值越接近 -1  → 相反
```

**在 NLP 中，余弦相似度是主流选择，原因见下一节。**

### 实际代码

```python
from sentence_transformers import SentenceTransformer

model = SentenceTransformer('all-MiniLM-L6-v2')

embeddings = model.encode(['汽车', '轿车', '香蕉', '卡车'])

# 计算所有词对之间的相似度
from sklearn.metrics.pairwise import cosine_similarity
sim = cosine_similarity(embeddings)

import pandas as pd
print(pd.DataFrame(sim,
    index=['汽车', '轿车', '香蕉', '卡车'],
    columns=['汽车', '轿车', '香蕉', '卡车']))
```

**预期输出**：
```
        汽车    轿车    香蕉    卡车
汽车   1.000   0.851   0.234  0.672
轿车   0.851   1.000   0.198  0.589
香蕉   0.234   0.198   1.000  0.156
卡车   0.672   0.589   0.156  1.000
```

发现：
- 汽车↔轿车 = 0.851（非常相似！）
- 汽车↔卡车 = 0.672（也比较相似，都是交通工具）
- 汽车↔香蕉 = 0.234（不相似）
- 香蕉↔卡车 = 0.156（最不相似）

## 5. Cosine Similarity 是什么

### 通俗解释

**Cosine Similarity = 看两个向量**"指的方向"**有多接近，忽略它们的长度。**

```
想象两个人从原点出发：

A 往东北方向走了 10 公里   → 向量的方向：东北
B 往东北方向走了 1 公里    → 向量的方向：东北

欧几里得距离说：差 9 公里，很远！（因为看绝对距离）
余弦相似度说：方向一样，非常相似！（因为看方向而非长度）
```

### 为什么 NLP 用余弦相似度而非欧几里得距离

**问题场景**：

```python
文本A："汽车"                          → Embedding 向量较短
文本B："一种用于道路交通的四轮机动车辆"  → Embedding 向量较长

# 两句话意思相同，但长度差很多
# 欧几里得距离：因为向量长度差异大，距离远 → 误判为"不相似"
# 余弦相似度：只看方向不看长度 → 正确判断为"相似"
```

在文本中，**同样意思的长文本和短文本，Embedding 向量的长度往往不同**。但它们的**方向**接近。所以 NLP 几乎都用余弦相似度。

### 计算过程（3 步）

```python
import numpy as np

def cosine_similarity(v1, v2):
    """一步一步计算余弦相似度"""
    # 第 1 步：点积（两个向量"在同一方向上的程度"）
    dot_product = np.dot(v1, v2)

    # 第 2 步：各自的长度
    norm1 = np.linalg.norm(v1)  # sqrt(sum(v1²))
    norm2 = np.linalg.norm(v2)

    # 第 3 步：点积除以长度的乘积
    cosine = dot_product / (norm1 * norm2)
    return cosine

# 取值范围：[-1, 1]
# 1.0  → 方向完全一致
# 0.0  → 方向无关（垂直）
# -1.0 → 方向完全相反
```

### 用熟悉的 Python 结构理解

```python
# 余弦相似度 ≈ 把向量想象成 Python 的 list
# 然后看它们对应位置的元素，是不是"同正同负"

vec_汽车 = [0.9, 0.1, 0.8]
vec_轿车 = [0.8, 0.1, 0.7]
vec_香蕉 = [0.1, 0.9, 0.1]

# 汽车 vs 轿车：每个位置都是正数，且数值接近 → 相似度高
# 汽车 vs 香蕉：第一个位置 0.9 vs 0.1 → 方向相反 → 相似度低
```

## 6. Semantic Search 是什么

### 通俗解释

**传统搜索（关键词匹配）**：
```
搜索："汽车"
结果：完全包含"汽车"二字的文档
问题：找不到"轿车"相关的内容，即使意思一样
```

**语义搜索（Embedding 搜索）**：
```
搜索："汽车"
1. 把"汽车"转成向量
2. 把所有文档转成向量
3. 找和"汽车"向量最接近的 K 个文档
4. 结果：包含"轿车""SUV""跑车"等语义相关的内容
```

### 工作流程

```
文档库：[doc1, doc2, doc3, ..., docN]
   ↓
[Embedding 模型]
   ↓
向量库：[vec1, vec2, vec3, ..., vecN]  (N × 384)
   ↓
用户查询："汽车"
   ↓
[Embedding 模型]
   ↓
查询向量：query_vec (1 × 384)
   ↓
余弦相似度：query_vec 与每个 vec 比较
   ↓
排序 → 取 Top K
   ↓
返回相似度最高的 K 个文档
```

### 实际代码

```python
# 文档库
documents = [
    "汽车是一种交通工具",
    "轿车通常有四门五座",
    "SUV 有更高的底盘",
    "香蕉是一种热带水果",
    "苹果富含维生素 C",
    "卡车用于货物运输",
]

# Step 1: 把所有文档转成向量
doc_vectors = model.encode(documents)

# Step 2: 把查询转成向量
query = "汽车"
query_vector = model.encode([query])

# Step 3: 计算余弦相似度
from sklearn.metrics.pairwise import cosine_similarity
scores = cosine_similarity(query_vector, doc_vectors)[0]

# Step 4: 排序，取 Top K
top_indices = scores.argsort()[::-1][:3]

for i in top_indices:
    print(f'[{scores[i]:.3f}] {documents[i]}')
```

**预期输出**：
```
[1.000] 汽车是一种交通工具
[0.723] 轿车通常有四门五座
[0.618] SUV 有更高的底盘
```

注意："轿车"和"SUV"虽然没有"汽车"二字，但被语义搜索找到了！**这就是 Embedding 的力量。**

## 7. 可视化：在空间中看语义

### 为什么需要降维

真实的 Embedding 是 384 维甚至 768 维的——人脑无法直接"看见"384 维空间。

我们需要把 384 维**压缩到 2 维**（就像把 3D 地球仪压扁成 2D 地图），以便在平面上画出来。

常用降维算法：
- **PCA**：找一个"最能体现差异的视角"投影
- **t-SNE**：保持"近的近，远的远"的关系

### 可视化代码

```python
from sklearn.decomposition import PCA
import matplotlib.pyplot as plt

# 1. 生成不同主题的词
words = ['汽车', '轿车', 'SUV', '卡车', '巴士',          # 交通工具
         '香蕉', '苹果', '橘子', '葡萄', '西瓜',          # 水果
         'Python', 'Java', 'Go', 'Rust', 'JavaScript']    # 编程语言

vectors = model.encode(words)  # (15, 384)

# 2. PCA 降维到 2D
pca = PCA(n_components=2, random_state=42)
vectors_2d = pca.fit_transform(vectors)  # (15, 2)

# 3. 画散点图
colors = {'交通': '#FF6B6B', '水果': '#4ECDC4', '编程': '#45B7D1'}
labels = ['交通']*5 + ['水果']*5 + ['编程']*5

for i, word in enumerate(words):
    x, y = vectors_2d[i]
    plt.scatter(x, y, c=colors[labels[i]], s=100, edgecolors='white', linewidth=1)
    plt.annotate(word, (x, y), textcoords="offset points", xytext=(0, 10), fontsize=10)

plt.title('词向量 2D 可视化（PCA 降维）')
plt.show()
```

**你会看到**：同一类别的词在 2D 平面上**聚在一起**——交通工具在一边，水果在一边，编程语言在一边。它们之间的"距离"代表了语义相似度。

## 8. 总结

```
起点问题：
  "为什么 AI 知道'汽车'和'轿车'意思接近？"
      ↓
答案：
  因为它们在训练大数据的上下文中频繁同现。
      ↓
Embedding：
  把这种"语义关系"编码成了数字向量。
      ↓
向量：
  高维空间中的坐标。（384 个数字 = 384 个维度的坐标）
      ↓
相似度：
  两个向量有多"近"。近 = 语义相似。
      ↓
Cosine Similarity：
  看"方向"而不看"长度"的相似度指标。
  NLP 首选用它。
      ↓
Semantic Search：
  用 Embedding 做搜索。
  "汽车"能找到"轿车"，即使没出现这两个字。
```

### Embedding 的 4 个核心应用

| 应用 | 做什么 | 举例 |
|------|--------|------|
| 语义搜索 | 按意思搜，不按关键词 | "代步工具"搜出"汽车""自行车" |
| 文本聚类 | 把相似文本分到一组 | 自动分类客服工单 |
| 推荐系统 | "你还可能喜欢..." | 看了"汽车"文章的也看"轿车" |
| 异常检测 | 找到"不对劲"的内容 | 检测代码中不安全的模式 |
