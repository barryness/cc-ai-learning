## 流程

### 1. 分析被测代码
- 读取函数签名和类型提示
- 识别分支逻辑（if/else、循环、异常）
- 识别边界条件（空输入、极值、错误状态）

### 2. 生成测试用例
对每个函数生成：
- **正常用例**: 典型输入，验证正常行为
- **边界用例**: 空值、零值、极限值
- **异常用例**: 错误输入，验证错误处理

### 3. 遵循项目测试规范
- 使用项目指定的测试框架
- 遵循项目命名约定（test_xxx）
- 使用 fixture 和 parametrize 减少重复

## 输出格式

```python
# 每个测试函数包含 Given/When/Then 注释
def test_function_name_scenario():
    # Given: 准备测试数据
    ...
    # When: 执行被测函数
    ...
    # Then: 验证结果
    ...
```

## 覆盖率检查
- 行覆盖率目标: >= 80%
- 分支覆盖率目标: >= 70%
