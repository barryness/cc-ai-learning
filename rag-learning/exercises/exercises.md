练习1 [基础]：扩展知识库
─────────────────────────────────────────────────────
在 minimal_rag.py 的 DOCUMENTS 列表中添加 3 篇新文档，
内容可以是你熟悉的任何领域（游戏、历史、科技等）。
然后针对新文档提问，观察检索和回答效果。

练习2 [基础]：理解 top_k 的影响
─────────────────────────────────────────────────────
修改 rag.ask(query, top_k=1) 和 rag.ask(query, top_k=5)，
观察同样的检索文档数和回答质量的关系。
思考：top_k 太大或太小的各自问题是什么？

练习3 [进阶]：替换检索算法
─────────────────────────────────────────────────────
当前使用 TF-IDF 检索。尝试在 SimpleRetriever.retrieve() 中
将 cosine_similarity 改为欧氏距离（Euclidean distance）。
提示：from sklearn.metrics.pairwise import euclidean_distances
思考：相似度算法对检索结果有什么影响？

练习4 [进阶]：增加对话历史
─────────────────────────────────────────────────────
修改 generate_answer() 函数，增加对多轮对话的支持。
要求：能记住前一轮的问题和回答，在下一轮回答时可以引用上下文。
提示：在 messages 中添加 {"role": "assistant", "content": previous_answer}

练习5 [进阶]：替换为真实文档
─────────────────────────────────────────────────────
将 DOCUMENTS 替换为真实的文件读取。
- 读取一个 .txt 或 .md 文件
- 将文件内容按段落拆分为多个文档
- 用 RAG 检索+问答

练习6 [实战]：实现混合检索
─────────────────────────────────────────────────────
在 production/advanced_rag.py 中：
1. 增加 BM25 关键词检索（pip install rank-bm25）
2. 将 BM25 结果 + 向量检索结果融合（RRF: Reciprocal Rank Fusion）
3. 对比纯向量检索和混合检索的效果差异

练习7 [实战]：评估 RAG 效果
─────────────────────────────────────────────────────
设计一个简单的评估方案：
1. 准备 10 个问答对（问题 + 标准答案）
2. 用 RAG 生成答案
3. 对比生成答案和标准答案的相似度
提示：可以用 BLEU、ROUGE 或直接让 LLM 打分

练习8 [思考]：RAG 的局限性
─────────────────────────────────────────────────────
思考并写下你的理解：

3. 如果知识库太大（百万级文档），检索怎么加速？
